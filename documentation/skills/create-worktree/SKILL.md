# Create Worktree

This skill provides a deterministic workflow for creating or attaching Git worktrees without bypassing the repository's branch naming rules.

## When to Use This Skill

- When the user asks to create a worktree
- When the user wants a parallel checkout for a new task
- When a compliant branch name must be derived before worktree creation
- When you need to validate a branch name before running Git worktree commands
- When you want to avoid ad hoc `git worktree add` usage

## Prerequisites

- Run from anywhere inside this repository
- `pnpm` must be available so the branch validator can run
- The repository must already have the intended base branch locally, usually `main`

## Branch Naming Rules

Worktree branches must use the same format enforced elsewhere in the repository:

```text
<type>/<scope>-<description>
```

- Valid types:

<!-- types-start -->
| Type | Description |
| ---- | ----------- |
| `feat` | A new feature or capability that adds value for users |
| `fix` | A bug fix that addresses a specific issue or problem |
| `docs` | Documentation, AGENTS.md, SKILL.md, README, and planning files |
| `test` | Adding or correcting unit, integration, or end-to-end tests |
| `refactor` | Code restructuring that neither fixes a bug nor adds a feature |
| `style` | Formatting, whitespace, or code structure changes with no semantic effect |
| `perf` | A code change that improves performance (caching, query optimization, etc.) |
| `chore` | Housekeeping that doesn't modify src or test files (gitignore, editor config, etc.) |
| `ci` | GitHub Actions workflows, composite actions, and CI/CD scripts |
| `build` | Build system, Vite/Docker/Helm config, or external dependency integration |
| `revert` | Reverts a previous commit |

## Default Path Convention

Unless the user asks for a different location, create worktrees in a sibling directory named after the repository:

```text
../monorepo-worktrees/<branch-name-with-slashes-replaced-by-hyphens>
```

For example:

```text
../monorepo-worktrees/feat-lexico-user-auth
```

## Preferred Script

Use the bundled helper script instead of raw Git commands:

- Script: [create-worktree.sh](SKILL_DIR/scripts/create-worktree.sh)

### Script Usage

```bash
bash documentation/skills/create-worktree/scripts/create-worktree.sh <branch-name> [base-branch] [worktree-path]
```

### Examples

```bash
bash documentation/skills/create-worktree/scripts/create-worktree.sh feat/lexico-user-auth
bash documentation/skills/create-worktree/scripts/create-worktree.sh fix/conformance-generator-path main
bash documentation/skills/create-worktree/scripts/create-worktree.sh docs/monorepo-worktree-skill main ../sandbox/docs-worktree
```

## Workflow

### 1. Determine the branch name

- If the user already provided a compliant branch name, reuse it
- Otherwise derive `<type>/<scope>-<description>` from the requested task or current diff
- Prefer higher-level scopes such as `monorepo`, `applications`, or `packages` when the work spans multiple areas

### 2. Validate before creating anything

Run:

```bash
pnpm exec validate-branch-name -t "<branch-name>"
```

Do not create the worktree if validation fails.

### 3. Create or attach the worktree

Use the helper script.

- If the branch already exists locally, the script attaches a new worktree to it
- If the branch does not exist, the script creates it from the base branch and then creates the worktree

### 4. Report the outcome

Return:

- The branch name used
- The base branch used
- The final worktree path

## Troubleshooting

| Issue | Cause | Solution |
| ---- | ---- | ---- |
| Branch name rejected | Invalid type, scope, or description | Re-derive the name and rerun `pnpm exec validate-branch-name -t` |
| Worktree path already exists | Directory already present | Remove or choose a different path |
| Branch already checked out elsewhere | Existing worktree uses the branch | Attach to the existing branch at a different path or remove the old worktree first |
| Base branch missing locally | Local branch not fetched | Fetch or create the base branch before rerunning the script |
